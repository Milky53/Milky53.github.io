现象：在浮动布局中，父元素的高度默认被子元素撑开，当子元素浮动后，其从文档流中脱离，将无法撑起父元素的高度，导致父元素的高度丢失，父元素下的元素会自动上移，导致页面布局混乱。

解决方法

将父元素高度写死

  • 缺点：容易导致内容溢出，或内容空洞

BFC ( Block Formatting Context ) 块级格式化环境

BFC是CSS中一个隐含的属性

为元素开启BFC：元素变成独立的布局区域

带你用最简单的方式理解最全面的BFC_哔哩哔哩_bilibili

  • 开启BFC的元素特点（解决的问题）：

    • 当前元素不会被浮动元素所覆盖（清除浮动问题）
    • 父元素开启BFC外边距不会和子元素重叠（解决外边距重叠问题：包括兄弟之间、父子之间）
    • 父元素开启BFC可以包含浮动的子元素（解决高度塌陷问题）

  • 开启BFC方式：（均为间接开启，都有些副作用）

    • 设置浮动

      float:left （宽度丢失）

    • 将元素设置为行内块元素

      display:inline-block (宽度丢失)

    • 将overflow设置为非visible的值 (最常用)

      • overflow：scroll
      • overflow: auto
      • overflow: hidden（最常用）

clear (完美方式)

不希望某个元素因为其它元素浮动的影响而改变位置，可以通过clear属性来清楚浮动元素对当前元素所产生的影响

  • 作用：清除浮动元素对当前元素所产生的影响

  • 可选值：

    • left：清除左侧浮动元素对当前元素的影响
    • right：清除右侧浮动元素对当前元素的影响
    • both：清楚两侧中最大影响的那一侧

  • 原理：设置clear以后，浏览器会自动为元素添加一个上外边距，使其位置不受其他元素影响
