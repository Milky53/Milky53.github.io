对象

什么是对象？

  • 多个数据的封装体
  • 用来保存多个数据的容器
  • 一个对象代表现实中的一个事物

为什么要用对象？

  • 统一管理多个数据

对象的组成

  • 属性：属性名（字符串，允许不写引号）和属性值（任意数据类型）组成
  • 方法：一种特别的属性（属性值是函数）

如何访问对象的内部数据？

  • .属性名 常用，但有时不能用

  • ['属性名'] 不常用，但任何情况下都能用

    • 什么时候必须用['属性名']

      • 属性名包含特殊字符：- 空格
      • 属性名不确定

对象的分类

内建对象

由ES标准定义的对象，在任何的ES的实现中都可以使用

比如Date、Math、String、Number、Boolean、Function、Object……

宿主对象

由JS的运行环境提供的对象，目前来说主要指由浏览器提供的对象

比如BOM、DOM（一组对象）

自定义对象

由开发任意自己创建的对象

  • 创建对象

    • var obj = new Object();

    • var obj = {

      ​ name:”猪八戒”,

      ​ age:28,

      ​ test:{name:”沙和尚”}

      };

  • 添加属性

    • obj.name = “孙悟空”;

    • 属性值可以是任何数据类型，也可是函数（即对象的方法）

      obj.sayName = function(){

      ​ console.log(obj.name)；

      };

      调用对象的方法：obj.sayName();

  • 删除属性

    • delete obj.name;

  • in 运算符：可检查一个对象中是否由指定的属性

    语法：”属性名” in 对象

    示例：console.log(“name” in obj);

  • for … in 语句：枚举对象中的属性

    语法：for(var 变量 in 对象) { }

    示例：for(var n in obj){

    ​ console.log(“属性名：”+n);

    ​ console.log(“属性值：”+obj[n]);

    ​ }

函数

什么是函数？

函数也是一种对象

  • 函数：具有特定功能的n条语句的封装体
  • 只有函数是可以执行的，其它类型的数据不能执行

为什么要用函数？

  • 提高代码复用
  • 便于阅读交流

如何定义函数？

  • 创建函数

    • var fun = new Function() 少用

    • 使用函数声明：

      function 函数名([形参1，形参2…形参N]){

      ​ 语句…

      }

      最后不用写分号

      函数内部还可以在声明函数

    • 使用函数表达式：（var 函数名 = 匿名函数）

      var 函数名 = function([形参1，形参2…形参N]){

      ​ 语句…

      }；

      注意结尾要写分号

  • 形参与实参

    • 声明形参相当于在函数内部声明了对于变量，但是没有赋值

    • 调用时指定实参，实参会赋值给函数中对应的形参

    • 调用函数时解析器不会检查实参类型、数量

    • 实参可以是一个对象，也可以是一个函数

      形参为匿名函数 (在开发中常见)

  • 返回值

    • return xxx；
    • 可以是任意的数据类型，也可以是对象，也可以是函数

如何调用（执行）函数？

  • test()：直接调用

  • obj.test()：通过对象调用

  • new test()：new调用

  • test.call(obj)/apply(obj)：相当于obj.test()，可以让一个函数成为指定任意对象的方法进行调用

回调函数

什么函数是回调函数？

  • 自己定义的、自己没有调用、但最终执行了（某个时刻或某个条件下）

常见的回调函数

  • dom事件回调函数（this是发生事件的dom元素）

  • 定时器回调函数（this是window）

  • ajax请求回调函数

  • 生命周期回调函数

立即执行函数（IIFE，Immediately-Invoked Function Expression）

立即执行函数：函数定义完，立即被调用（别名：匿名函数自调用）

立即执行函数往往只会执行一次

  • 作用：

    • 隐藏实现

    • 不会污染外部（全局）命名空间

    • 用它来编写js模块

作用域

作用域：指一个变量的作用范围

在JS中一共有两种作用域：全局作用域、函数作用域

全局作用域

直接在script标签中的JS代码，都在全局作用域

全局作用域在打开时创建，在页面关闭时销毁

全局作用域中有一个全局对象window，它代表浏览器的窗口

在全局作用域中：

​ 创建的变量都会作为window对象的属性保存

​ 创建的函数都会作为window对象的方法保存

函数作用域

调用函数时创建函数作用域，函数执行完毕后函数作用域销毁

每调用一次函数就会创建一个新的函数作用域，它们之间是相互独立的

在函数作用域中要访问同名的全局作用域属性a，可用window.a

对象创建模式

使用Object构造函数创建对象

  • 套路：先创建空Object对象，再动态添加属性/方法
  • 适用场景：起始时不确定对象内部数据
  • 问题：语句太多（使用对象字面量模式解决）

使用对象字面量创建对象

  • 套路：使用{}创建对象，同时指定属性/方法
  • 适用场景：起始时对象内部数据是确定的
  • 问题：如果创建多个对象，有重复代码（使用工厂模式解决）

使用工厂方法创建对象

批量创建对象

  • 套路：通过工厂函数创建对象并返回
  • 适用场景：需要创建多个对象
  • 问题：对象没有一个具体的类型，都是Object类型（使用自定义构造函数解决）

使用工厂方法创建的对象，使用的构造函数都是Object，所创建的对象都是Object类型，导致无法区分出多种不同类型的对象

使用自定义构造函数创建对象

  • 套路：自定义构造函数，通过new创建对象
  • 适用场景：需要创建多个类型确定的对象
  • 问题：每个兑现都有相同的数据，浪费内存（使用构造函数+原型解决）

构造函数和普通函数的区别就调用方式的不同

普通函数直接调用，构造函数需要使用new关键词调用

构造函数执行流程：

​ 1.立刻创建一个新的对象

​ 2.将新建的对象设置为函数中的this，在构造函数中可以使用this来引用新建的对象

​ 3.逐行执行函数中的代码

​ 4.将新建的对象作为返回值返回

使用同一个构造函数创建的对象，称为一类对象，也将一个构造函数称为一个类

将通过一个构造函数创建的对象称为该类的实例

使用instanceof可以检查一个对象是否是一个类的示例

  • 语法：对象 instanceof 构造函数
  • 示例：console.log(dog instanceof Person) False

构造函数每执行一次就会创建一个新的sayName方法，所有实例的sayName都是唯一的。为了提高性能，可将sayName在全局作用域定义。

但将函数定义在全局作用域，污染了全局作用域的命名空间，且不安全（容易被覆盖）

因此可以将将sayName在原型对象里定义

构造函数+原型的组合模式

  • 套路：自定义构造函数，属性再函数中初始化，方法添加到原型上
  • 适用场景：需要创建多个类型确定的对象

原型对象

我们所创建的每一个函数，解析器都会向函数中添加一个属性prototype，这个属性对应着一个对象，这个对象称原型对象

  • 若函数以普通函数的形式调用，prototype没有任何作用
  • 若函数以构造函数的形式调用，它所创建的对象中都会有一个隐含的属性，指向该构造函数的原型对象，可通过___proto__来访问该属性

原型对象就相当于一个公共区域，所有同一个类的实例都可以访问到这个原型对象。

因此，创建构造函数时，可将对象中共有的属性和方法统一设置到原型对象中。（不用为每个对象创建属性，也不会影响到全局作用域）

当访问对象的一个属性或方法时，会先在对象自身中寻找，若没有则会去原型对象中寻找。

（若再没有则去原型的原型中寻找，直到找到Object对象的原型，Object对象的原型没有原型，如果在Object中依然没有找到，则返回undefined。

使用in检查对象中是否有某属性时，如果对象中没有但原型中有，也会返回true

可使用对象的hasOwnProperty()来检查对象自身中是否有该属性
