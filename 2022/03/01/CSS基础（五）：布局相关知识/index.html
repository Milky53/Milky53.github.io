文档流（normal flow）

网页是一个多层结构，用户只能看到最顶层

最底下的一层称为文档流，文档流是网页的基础

创建的元素默认都在文档流中进行排列

元素主要有两个状态：

  • ​ 在文档流中

  • ​ 不在文档流中（脱离文档流）

元素在文档流中的特点

  1. 块元素

    • 总是独占一行（无论多宽，自上向下垂直排列）
    • 默认宽度：父元素的全部（即把父元素撑满）
    • 默认高度：被子元素（子元素）撑开
    • 比如：div

  2. 行内元素

    • 不会独占一行，只占自身大小（从左到右水平排列）
    • 若一行中不能容纳下所有行内元素，则换行（与书写习惯一致）
    • 默认宽度：被子元素（子元素）撑开
    • 默认高度：被子元素（子元素）撑开
    • 设置宽高不能生效

一旦块元素和行内元素脱离文档流，则不具备以上特点

盒子模型（box model）

所有元素都是一个矩形的盒子

每个盒子都由几个部分组成：

  • 内容区（content）：内容区大小由width和height决定
  • 内边距（padding）：会影响盒子大小
  • 边框（border）：会影响盒子大小
  • 外边距（margin）

盒子的可见框的大小，默认情况下由内容区、内边距、边框共同决定

盒子的实际大小，还受外边距影响

边框（border）

  1. 边框宽度：border-width

    • 可只指定四个方向的边框宽度（顺时针）

    • 如果省略，一般为3px

    • 四个值：上 右 下 左

    • 三个值：上 左右 下

    • 两个值： 上下 左右

    • 一个值：上下左右

    • 示例：

    • border-xxx-width：

      xxx可以是 top right bottom left

  2. 边框颜色：border-color

    • 可只指定四个方向的边框颜色：规则同上
    • 如果省略，默认使用color的值（color默认为black）

  3. 边框样式：border-style

    • 可只指定四个方向的边框样式：规则同上
    • solid 实线、dotted 点状虚线、dashed 虚线、double 双线
    • 不可省略，默认值为none

  4. border简写属性（最常用）

    • border：width color style; (可改变顺序)

    • 示例：

    • border-xxx 同理

      xxx可以是 top right bottom left

  5. 边框圆角：border-radius

    • border-radius

      • 四个值：左上 右上 右下 左下
      • 三个值：左上 右上/左下 右下
      • 两个值：左上/右下 右上/左下
      • 一个值：四个角

    • border-top-left-radius

    • border-top-right-radius

    • border-bottom-left-radius

    • border-bottom-right-radius

内边距（padding）

内边距：内容区和边框之间的距离

内边距的设置会影响到盒子的大小

背景颜色会延伸到内边距上

  • padding-top
  • padding-right
  • padding-bottom
  • padding-left
  • 简写padding:
    • 规则同border-width

外边距（margin）

外边距不会影响盒子可见框大小

但是会影响盒子的位置（即影响盒子实际大小）

  • margin-top
    • 上外边距，设置一个正值，元素（自身）会向下移动
  • margin-right
    • 默认情况下，不会产生任何效果；总是被自动调整
  • margin-bottom
    • 下外边距，设置一个正值，其下边的元素会向下移动
  • margin-left
    • 左外边距，设置一个正值，元素（自身）会向右移动

负值则方向相反

  • 简写margin:
    • 规则同border-width
    • margin: 0px auto (水平居中)

外边距重叠问题（垂直外边距的重叠）

相邻的垂直方向的外边距会发生重叠现象

兄弟元素间的垂直外边距

  • 两者都是正值：取两者之间的较大值
  • 特殊情况：
    • 一正一负：取两者之和
    • 都负：取绝对值较大值
  • 兄都元素之间的外边距重叠对开发有利，不需要处理

盒子的大小

盒子的可见框大小

  • 默认情况：由内容区、内边距、边框共同决定
  • box-sizing：用来设置盒子尺寸的计算方式，即设置width和height的作用
    • 可选值：
      • content-box：默认值，宽度和高度用来设置内容区的大小
      • border-box：宽度和高度用来设置整个盒子的可见框大小（width和height指的是内容区和边框的总大小，不包括外边距）

盒子的实际大小

  • 还受外边距影响

盒子的水平布局 （等式）

元素在其父元素中水平方向的位置由一下几个属性共同决定：

  • margin-left
  • border-left
  • padding-left
  • width
  • padding-right
  • border-right
  • margin-right

一个元素在其父元素中水平布局必须要满足以下式：

margin-left + border-left + padding-left + width + padding-right + border-right + margin-right = 其父元素内容区的宽度

若相加结果使等式不成立，称为过度约束，则等式会自动调整：

  • 如果这七个值中没有为auto的情况：

    则浏览器会自动调整margin-right值以使等式满足

  • 这七个值中有三个可以设置为auto：

    • width

    • margin-left

    • margin-right

      (padding不可以自动调整)

    若某个值为auto，则自动调整设有auto的那个属性使等式成立

    （width的值默认是auto）

    若将width和一个外边距设置为auto，则width调整为最大，设置为auto的外边距会自动为0

    若将两个外边距设置为auto，width固定，则外边距设置为相同值（居中）

    若将三个值都设置为auto，则width调整为最大，外边距都为0

盒子的垂直布局 （溢出问题）

默认情况下父元素的高度被内容撑开

overflow （用于解决溢出）

  • 溢出：子元素的大小超过了父元素，则子元素从父元素中溢出
  • 解决溢出：使用overflow属性设置父元素
  • 可选值：
    • visible（默认值）：溢出
    • hidden：溢出的内容被裁剪不会显示
    • scroll：生成两个（垂直+水平）滚动条，通过滚动条查看完整内容
    • auto：根据需要生成滚动条
  • overflow-x：单独处理水平
  • overflow-y：单独处理垂直

以上所有内容以块元素为例(div)

行内元素的盒模型

行内元素不支持设置宽度和高度，内容区大小由内容直接决定

行内元素可以设置padding，但是垂直方向padding不会影响页面的布局

行内元素可以设置border，但是垂直方向border不会影响页面的布局

行内元素可以设置margin，但是垂直方向margin不会影响页面的布局

display（块元素和行内元素相互转换）

  • 设置元素显示的类型（块元素和行内元素相互转换）
  • 可选值：
    • inline：设置为行内元素
    • block：设置为块元素
    • inline-block：设置为行内块元素（开发时不常用）
      • 既可以设置宽度和高度，又不会独占一行
    • table：设置为一个表格
    • none：不显示（隐藏），不占据页面位置
  • 应用：下拉框none—>block

visibility

  • 设置元素显示的状态
  • 可选值：
    • visible：默认值，元素正常显示
    • hidden：不显示（隐形），但是依然占据页面的位置

轮廓（outline）

与border区分

  • 不同点：轮廓不会影响到可见框的大小，不影响页面布局

outline多应用于鼠标悬停时：

阴影（box-shadow)

  • 不影响页面布局

  • 语法：box-shadow：水平偏移量 垂直偏移量 模糊半径颜色

    • 水平偏移量：设置阴影水平位置，正值向右移动，负值相反
    • 垂直偏移量：设置阴影垂直位置，正值向下移动，负值相反
    • 模糊半径：越大越模糊
    • 颜色：rgba(0，0，0，.3)

浏览器的默认样式

浏览器会为元素设置默认样式，开发时需要去除默认样式（pc端）

去除默认样式

最完美的写法，通过引入重置样式表

  • reset.css：直接去除浏览器的默认样式
  • normalize.css：对默认样式进行了统一
